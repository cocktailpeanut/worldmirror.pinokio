# WorldMirror 2.0

Pinokio launcher for the currently released WorldMirror 2.0 demo from Tencent Hunyuan's HY-World 2.0 project.

As of the upstream HY-World 2.0 README entry dated April 16, 2026, only the WorldMirror 2.0 reconstruction code and weights are open-sourced. The full text/image-to-world generation pipeline is still marked "Coming Soon". This launcher therefore packages the released WorldMirror 2.0 Gradio app.

## What This Launcher Does

- Clones the official `Tencent-Hunyuan/HY-World-2.0` code into `app/`
- Installs the released WorldMirror 2.0 Python dependencies into `app/env`
- Starts the WorldMirror 2.0 Gradio UI on `127.0.0.1` with an automatic Pinokio port
- Lets you upload multi-view images or videos and export 3D outputs such as point clouds, Gaussian splats, depth maps, normals, and camera parameters

## Requirements

- NVIDIA GPU recommended by upstream
- CUDA 12.4 recommended by upstream
- Python 3.10 environment managed by Pinokio
- `flash-attn` must install successfully because the released upstream code imports it directly
- On Windows, this launcher installs the matching prebuilt wheel from `lldacing/flash-attention-windows-wheel` on Hugging Face for `torch==2.4.0`, CUDA 12.4, and Python 3.10

The upstream repository also splits some binary dependencies by platform. This launcher normalizes that by using Windows/Linux-specific `gsplat` and `onnxruntime` selections in `requirements.pinokio.txt`.

## How To Use

1. Open the app in Pinokio and click `Install`.
2. Wait for dependency installation to finish. Model weights download automatically from Hugging Face on first run.
3. Click `Start WorldMirror 2.0`, or use `Start WorldMirror 2.0 Low VRAM (bf16, 640)` on smaller GPUs.
4. In the Gradio UI, upload a video or a set of images, then click `Reconstruct`.
5. Use `Open Outputs` in the launcher after a run to inspect `app/gradio_demo_output`.

For lower VRAM usage, prefer the `Start WorldMirror 2.0 Low VRAM (bf16, 640)` menu item and increase the Gradio upload sampling interval so fewer video frames are extracted before reconstruction.

## Files And Outputs

- Upstream code: `app/`
- Example scenes: `app/examples/worldrecon`
- Generated reconstructions: `app/gradio_demo_output`
- Camera parameters: downloaded from the `Camera Parameters` tab in the UI
- Gaussian splats / point clouds / meshes: produced per run inside the output directory

## Programmatic Access

### Python

Upstream ships a direct Python API:

```python
from hyworld2.worldrecon.pipeline import WorldMirrorPipeline

pipeline = WorldMirrorPipeline.from_pretrained("tencent/HY-World-2.0")
result_dir = pipeline("path/to/images")
print(result_dir)
```

You can also use the upstream CLI from the app folder:

```bash
python -m hyworld2.worldrecon.pipeline --input_path path/to/images
```

### JavaScript

For scripted local batch runs, call the upstream CLI from Node.js:

```javascript
import { spawn } from "node:child_process";

const proc = spawn(
  "python",
  ["-m", "hyworld2.worldrecon.pipeline", "--input_path", "path/to/images"],
  { cwd: "./app", stdio: "inherit" }
);

proc.on("exit", (code) => {
  console.log("WorldMirror 2.0 finished with code", code);
});
```

If you want to automate the running web UI instead, Gradio exposes a generated API page. After startup, open the footer's `Use via API` link or inspect `http://127.0.0.1:<PORT>/gradio_api/openapi.json`. Gradio's docs state that unnamed endpoints default to the Python function name, so the main reconstruction handler is exposed from the `gradio_demo` callback in the upstream app.

### Curl

Inspect the autogenerated Gradio OpenAPI schema:

```bash
curl http://127.0.0.1:<PORT>/gradio_api/openapi.json
curl http://127.0.0.1:<PORT>/openapi.json
```

Replace `<PORT>` with the actual running port shown in the launcher URL. Use whichever schema path your running Gradio build exposes, or fall back to the web UI's `Use via API` page to confirm the exact request shape before sending automated HTTP requests.
